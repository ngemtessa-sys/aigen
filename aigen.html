06–07 / UI & User Flows

Key Scenario Walkthroughs

01
Register a New Model
Developer
  • 1Navigate to Models → New Model
  • 2Enter name, provider, model type, and version
  • 3Paste or fill JSON configuration (temperature, max_tokens…)
  • 4Save → model created with status DEVELOPMENT
  • 5Audit log entry recorded automatically
02
Create a Prompt Template
Developer
  • 1Navigate to Prompt Library → New Template
  • 2Enter name, description, and template body with {{variable}} placeholders
  • 3Define variables schema (name, type, default)
  • 4Save → version 1 created; previous versions preserved
  • 5Template immediately available in playground
03
Test a Prompt in Playground
Developer
  • 1Open any prompt template → Playground tab
  • 2Fill in variable values in the sidebar form
  • 3Select target model from dropdown
  • 4Click Run → live inference call made
  • 5Output shown alongside latency, token count, and cost
04
Generate an API Key
Developer
  • 1Navigate to API Keys → Generate Key
  • 2Name the key and select scopes (inference, read, admin)
  • 3Optionally set an expiry date
  • 4System generates key — raw value shown ONCE only
  • 5Subsequent views show only the 8-character prefix
05
Monitor Usage Analytics
Business Analyst
  • 1Navigate to Usage Analytics
  • 2Set date range (last 7d / 30d / custom)
  • 3Filter by model or prompt template
  • 4Review requests, token consumption, cost, error rate
  • 5Export CSV for reporting
06
Set Up an Alert
Workspace Admin
  • 1Navigate to Alerts → New Alert
  • 2Name the alert and define condition (e.g. error_rate > 5%)
  • 3Set evaluation window (e.g. 10-minute rolling)
  • 4Add notification channels: email or Slack webhook
  • 5Enable and save — alert fires when threshold is crossed
07
Submit a Fine-tuning Job
ML Engineer
  • 1Navigate to Fine-tuning → New Job
  • 2Select base model, upload JSONL dataset to S3
  • 3Configure hyperparameters (epochs, learning rate, batch size)
  • 4Submit — job status: PENDING → RUNNING → COMPLETE
  • 5Result model auto-registered in Model Registry
08
Promote Model to Production
Workspace Admin
  • 1Open Model detail view → Versions tab
  • 2Select a version in STAGING status
  • 3Click Promote to Production — confirmation dialog shown
  • 4Status updated; inference traffic routes to new version
  • 5Audit log entry created with timestamp and actor

10 / Risks & Recommendations

Known Risks & Mitigations

🔴 High — Data Privacy
Sensitive Data in Prompts
Prompts and model outputs may contain PII or sensitive business data stored in inference logs.
✓ MITIGATION
Field-level encryption for input/output columns. PII anonymization pipeline on ingest. Configurable retention policies per workspace.
🟠 Medium — Cost
Runaway API Cost Overruns
Unthrottled inference calls can generate unexpected provider API costs at scale.
✓ MITIGATION
Real-time token usage alerts. Per-API-key rate limiting via Redis. Cost estimation shown in playground before submission.
🟠 Medium — Reliability
Model Performance Variability
Provider outages or silent model degradation can impact end users without detection.
✓ MITIGATION
Circuit breaker pattern on inference client. Automatic fallback model routing. Error rate alerts with sub-5-minute detection.
🟣 Critical — Security
Multi-Tenant Isolation Breach
Cross-workspace data leakage would be a critical security incident with legal consequences.
✓ MITIGATION
Strict Postgres RLS policies on every table. Automated RLS audit tests in CI. Penetration testing before Phase 3 launch.
🔵 Medium — Performance
Inference Log Volume
High-traffic workspaces can generate millions of InferenceLog rows, degrading query performance.
✓ MITIGATION
TimescaleDB hypertable with time partitioning. Continuous aggregates for analytics. Archiving policy for logs over 90 days.
🔵 Low — Complexity
Integration Complexity
Each AI provider has different APIs, auth patterns, and response formats requiring ongoing maintenance.
✓ MITIGATION
Unified adapter interface built in Sprint 0. Providers added behind feature flags. Each adapter has its own integration test suite.
11 / AI Feature Suggestions

AI Capabilities Within Aigen

Beyond managing external AI, Aigen applies AI internally to improve its own usability — surfacing insights that would otherwise require manual analysis.

✨
Prompt Optimization Suggestions
Analyses historical inference logs to suggest edits to prompt templates that correlate with lower latency, fewer errors, and higher user ratings. Surfaced as inline suggestions in the Prompt Library editor.
🔍
Anomaly Detection in Usage Patterns
Statistical baseline model (rolling average + standard deviation) flags unusual spikes in request volume, error rate, or cost — generating alerts automatically without manual threshold configuration.
📈
Cost Forecasting
Time-series forecasting (Prophet or ARIMA) projects token spend for the next 7 and 30 days based on growth trends. Shown as a confidence band chart on the Analytics dashboard.
🎯
Model Recommendation Engine
Given a task description (classification, summarisation, RAG, etc.), suggests the best registered model based on historical performance data and cost-per-token trade-offs.
🧬
Fine-tuning Hyperparameter Advisor
Before job submission, an LLM-powered advisor analyses dataset size and task type, then recommends epoch count, learning rate, and batch size with reasoning — reducing trial-and-error cycles significantly.

Sample API Route — POST /models

TypeScript — app/api/workspaces/[workspaceId]/models/route.ts 
import { NextRequest, NextResponse } from 'next/server';
import { prisma } from '@/lib/db';
import { getSession } from '@/lib/auth';
import { modelCreateSchema } from '@/lib/validations';

export async function POST(
  req: NextRequest,
  { params }: { params: { workspaceId: string } }
) {
  const session = await getSession();
  if (!session)
    return NextResponse.json({ error: 'Unauthorized' }, { status: 401 });

  const membership = await prisma.workspaceUser.findUnique({
    where: { workspaceId_userId: {
      workspaceId: params.workspaceId, userId: session.userId
    }}
  });
  if (!membership || !['WORKSPACE_ADMIN','DEVELOPER'].includes(membership.role))
    return NextResponse.json({ error: 'Forbidden' }, { status: 403 });

  const body    = await req.json();
  const payload = modelCreateSchema.parse(body);

  const model = await prisma.model.create({
    data: { ...payload, workspaceId: params.workspaceId, createdBy: session.userId }
  });

  await prisma.auditLog.create({ data: {
    userId: session.userId, workspaceId: params.workspaceId,
    action: 'CREATE', resourceType: 'Model',
    resourceId: model.id, details: payload
  }});

  return NextResponse.json(model, { status: 201 });
}
